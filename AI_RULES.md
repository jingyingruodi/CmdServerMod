# AI 开发规范与项目准则 (v1.0)

为了防止项目管理再次陷入混乱，所有参与本项目的 AI 助手必须严格遵守以下规则：

## 1. 文档管理规则 🚫
- **禁止创建**：严禁创建任何形式的 `SUMMARY.md`, `LOG.md`, `DIAGNOSTIC.md`, `PLAN.md` 或带有日期/版本后缀的临时文档。
- **唯一更新点**：
    - 所有的架构说明必须更新到 `DEVELOPMENT.md`。
    - 所有的用户说明必须更新到 `README.md`。
    - 所有的版本变更必须记录在 `CHANGELOG.md`。
- **清理逻辑**：如果发现过时的文档，应将其内容整合后在回复中建议用户手动物理删除，而不是继续在旧文档上堆砌内容。

## 2. 代码开发准则 💻
- **环境意识**：必须区分 `Dedicated Server` (无 UI 环境) 和 `Client/Host` (有 UI 环境)。
- **禁止暴力 Hook**：严禁直接覆盖全局 `Chat.Text` 等原始函数，应优先使用 `UI.Run` 拦截或逻辑层判断。
- **日志规范**：所有日志必须带有 `[CmdServerMod]` 前缀。
- **版本一致性**：修改代码后，必须同步检查 `def.json` 中的 `version_name` 和代码内部的打印版本号是否一致。

## 3. 目录结构规范 📁
- `data/`：存放 Simulation (逻辑层) 代码，负责命令解析和执行。
- `ui/`：存放 UI 层代码，负责拦截输入和显示反馈。
- `simulation/`：**已废弃**，禁止在此目录下编写新代码。

## 4. 交互建议 🗣️
- 在提交大规模修改前，先通过 `read_file` 读取 `def.json` 确认当前项目状态。
- 如果遇到 `.gitignore` 导致的读取失败，应先告知用户，并按用户提供的流程临时修改 `.gitignore`，操作完成后必须复原。
